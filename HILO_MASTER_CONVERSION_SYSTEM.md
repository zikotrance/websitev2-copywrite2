## 1.1 FRAMEWORK DEFINITIONS

Each framework provides a distinct lens for validating HILO strategy decisions. All conclusions in this document are backed by at least one framework.

---

### FRAMEWORK 1: MECLABS Conversion Heuristic

**Formula**: `C = 4M + 3V + 2(I-F) - 2A`

| Variable | Definition | Weight | Application to HILO |
|----------|------------|--------|---------------------|
| **C** | Conversion Rate | Output | Target: 4.5%+ (50% above industry) |
| **M** | Motivation | 4x (highest) | User's desire to solve timeline/delay problem |
| **V** | Value Proposition | 3x | Clarity of "Zero Delays" promise |
| **I** | Incentive | 2x | Urgency elements (capacity, lead time) |
| **F** | Friction | -2x | Form fields, steps, cognitive load |
| **A** | Anxiety | -2x | Trust deficit, credibility concerns |

**Core Principle**: Motivation and Value Proposition have the highest impact. Friction and Anxiety must be minimized before optimizing Incentive.

---

### FRAMEWORK 2: Hormozi Value Equation

**Formula**: `V = (Perceived Value × Probability of Success) / (Effort + Time to Result)`

| Variable | Definition | HILO Application |
|----------|------------|------------------|
| **Perceived Value** | Specific outcome desired | "$480K saved opening revenue" |
| **Probability** | Likelihood of success | "98% on-time delivery, 5-year warranty" |
| **Effort** | Work required to start | "5 minutes to timeline" |
| **Time** | Duration until result | "Instant estimate" |

**Core Principle**: To increase value, INCREASE numerator (specificity + probability) and DECREASE denominator (effort + time). Generic CTAs like "Get Quote" fail because they imply high effort and unknown time.

---

### FRAMEWORK 3: Dan Kennedy (No B.S. Marketing)

**Core Principles**:

| Principle | Definition | HILO Application |
|-----------|------------|------------------|
| **The List** | Target audience precision | Hotels 80+ rooms, Real Estate 100+ units |
| **The Offer** | Irresistible, risk-reversed | Zero-Delay Guarantee + 5-year warranty |
| **The Message** | Direct, benefit-focused | Pain-focused copy, no fluff |
| **One CTA** | Single response mechanism | Wizard as sole conversion goal |
| **No B.S.** | Every word earns its keep | Cut content 30%, remove generic copy |

**Core Principle**: Confusion kills conversion. One offer, one CTA, no fluff. Content over 500 words per page violates this principle.

---

### FRAMEWORK 4: Eugene Schwartz (Awareness Levels)

**5 Levels of Customer Awareness**:

| Level | State | Marketing Approach | HILO Target Pages |
|-------|-------|-------------------|-------------------|
| **1. Unaware** | Doesn't know problem exists | Create awareness | NOT target for B2B high-ticket |
| **2. Problem Aware** | Knows problem, not solutions | Agitate pain, introduce solution | SEO informational pages |
| **3. Solution Aware** | Knows solutions exist, not yours | Differentiate, show unique value | Industry pages (PRIMARY TARGET) |
| **4. Product Aware** | Knows your product, not convinced | Handle objections, prove value | Solution pages |
| **5. Most Aware** | Ready to buy, needs to act | Remove friction, enable action | Wizard, Retargeting |

**Core Principle**: Different awareness levels require DIFFERENT page structures. A single page cannot serve all levels optimally.

---

### FRAMEWORK 5: StoryBrand SB7 Framework

**7-Element Narrative Structure**:

| Element | Definition | HILO Implementation |
|---------|------------|---------------------|
| **1. Character** | Customer is the hero | Hotel PM, Developer, Facility Manager |
| **2. Problem** | External, internal, philosophical | Delays, stress, unfair situation |
| **3. Guide** | Brand has empathy + authority | "We've solved this 180+ times" |
| **4. Plan** | Clear 3-4 step process | Quote → Timeline → Production → Install |
| **5. CTA** | Direct call to action | "Lock Your Timeline →" |
| **6. Success** | What success looks like | "On time, reputation intact" |
| **7. Failure** | What failure looks like | "Generic suppliers: 40% delayed" |

**Core Principle**: Customers buy solutions to INTERNAL problems (stress, fear, reputation risk), not external ones. Missing internal/philosophical problem layers reduces conversion.

---

### FRAMEWORK 6: PAS Framework

**Copy Sequence Structure**:

| Stage | Definition | Length | HILO Application |
|-------|------------|--------|------------------|
| **P** (Problem) | State the problem clearly | 1-2 sentences | "40% of hotel projects face mirror delays." |
| **A** (Agitate) | Amplify emotional impact | 2-3 sentences | "Each week = $80K lost. Plus explaining to ownership." |
| **S** (Solution) | Present resolution | 1-2 sentences + CTA | "Hilo: 180+ projects, zero delays. Lock timeline →" |

**Core Principle**: 3 separate problem cards dilute impact. ONE focused PAS block outperforms 3 diluted cards by reducing cognitive load and increasing emotional resonance.

---

### FRAMEWORK 7: AIDA Model

**Page Flow Structure**:

| Stage | Definition | HILO Block Position |
|-------|------------|---------------------|
| **A** (Attention) | Capture immediate focus | Block 1: Hero H1 |
| **I** (Interest) | Engage with relevant details | Blocks 2-3: Solution + Problem |
| **D** (Desire) | Build emotional want | Blocks 4-5: Case Study + Comparison |
| **A** (Action) | Enable the response | Block 6+: CTA |

**Core Principle**: Each page must flow through AIDA stages in order. Misplaced elements (e.g., comparison table buried at Block 10) break the sequence and reduce conversion.

## 1.3 CONFLICTS IDENTIFIED BETWEEN FRAMEWORKS

### CONFLICT 1: Solution vs Problem Order

| Framework | Position | Reasoning |
|-----------|----------|-----------|
| **MECLABS** | Trust → Problem → Solution | Anxiety must be addressed before offer |
| **StoryBrand** | Problem → Guide → Plan | Character has problem, then meets guide |
| **Schwartz (Level 3)** | Solution/Differentiation FIRST | They already know solutions exist |
| **PAS** | Problem → Agitate → Solution | Standard sequence |
| **AIDA** | Attention → Interest → Desire | Problem = Interest, Solution = Desire |

**RESOLUTION**:

| Page Type | Target Awareness | Recommended Order | Framework Rationale |
|-----------|------------------|-------------------|---------------------|
| **Homepage** | Mixed (Level 2-3) | Pain → Solution → Proof → CTA | PAS + StoryBrand: Establish problem first for mixed audience |
| **Industry Pages** | Level 3 (Solution Aware) | Solution/Differentiation → Pain → Proof → CTA | Schwartz Level 3: They know solutions; differentiate first |
| **Solution Pages** | Level 3-4 | Proof → Objection Handling → CTA | Schwartz Level 4: They know you; prove value |
| **Retargeting** | Level 4-5 | CTA → Reduced Form | Schwartz Level 5: Enable immediate action |

---

### CONFLICT 2: Trust Bar Position

| Framework | Position | Reasoning |
|-----------|----------|-----------|
| **MECLABS** | BEFORE Problem | Anxiety must be addressed before amplifying pain |
| **StoryBrand** | AFTER Problem (Guide element) | Character has problem, THEN meets guide |
| **AIDA** | Attention or Desire stage | Can be either |

**RESOLUTION**: Trust Bar INTEGRATED into Hero section (not separate block). This satisfies MECLABS (visible immediately) while maintaining StoryBrand sequence (guide authority present from start).

**Implementation**: Hero = Pain H1 + Trust Sub-element ("Trusted by 180+ hospitality projects") + CTA. Trust Bar remains but is condensed into single line within Hero.

---

### CONFLICT 3: Aggressive vs Consultative Tone

| Framework | Tone | Reasoning |
|-----------|------|-----------|
| **Kennedy** | Direct, aggressive | "No B.S.", one clear offer |
| **StoryBrand** | Empathetic, guiding | Brand as guide, customer as hero |
| **Hormozi** | Outcome-focused | Specific value promise |

**RESOLUTION**: Use **Kennedy directness** in structure (one CTA, minimal content, no fluff) but **StoryBrand empathy** in copy tone ("Here's the deal", "We know what's at stake").

**Implementation**:
- Structure: Kennedy (concise, one CTA, no distractions)
- Copy: StoryBrand (empathetic, customer-focused, narrative)
- CTA: Hormozi (outcome + time + specificity)
