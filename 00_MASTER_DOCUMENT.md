## 1. Vue d'Ensemble du Projet

### Contexte

HILO est un fournisseur B2B de produits en verre et miroirs pour les secteurs hospitality, multi-family, senior living, et commercial. L'optimisation du site web vise à maximiser les conversions (leads qualifiés) via une stratégie de contenu orientée conversion.

### Objectifs

| Objectif | KPI |
|----------|-----|
| **Génération de leads** | Taux de conversion > 5% |
| **Qualification** | Leads qualifiés > 70% |
| **Réduction friction** | Temps de complétion wizard < 90s |

### Public Cible

- **Primaire**: Développeurs immobiliers, gestionnaires hôtels, directeurs facilities
- **Secondaire**: Architectes, entrepreneurs généraux, designers d'intérieur
- **Décideurs**: B2B, cycle de vente moyen-long

### Différentiateurs Concurrentiels

1. **Fiabilité documentée** — 0.3% failure rate sur 5 ans
2. **Garantie 5 ans complète** — Sans pro-rating
3. **Prix compétitif direct** — Modèle sans intermédiaire
4. **AI estimation instantanée** — 60 secondes, sans appel commercial

## 2. Les 7 Critères HILO

### ⚠️ CRITIQUE — À appliquer sur TOUTES les pages

Ces 7 critères constituent le cœur de la proposition de valeur HILO. Ils doivent être présents de manière explicite ou implicite sur chaque page du site.

### Ordre de Priorité

| Priorité | Critère | Mots-clés | Application Obligatoire |
|----------|---------|-----------|------------------------|
| **#1** | Fiabilité long terme | 5 ans sans bris, <0.5% failure rate | Hero subheadline OU section dédiée |
| **#2** | Prix compétitif | Competitive pricing, volume discounts | Section pricing OU callout box |
| **#3** | Delivery porte-à-porte | Door-to-door, damage-free guarantee | Section shipping OU mentionnée |
| **#4** | Flexibilité | Catalogue ou sur mesure | Decision box quand applicable |
| **#5** | Minimum interaction | AI estimation instantanée | CTA "Get AI Estimate" visible |
| **#6** | Grand choix | 200+ produits, gamme complète | Cross-sell vers autres catégories |
| **#7** | Installation | Via partenaires certifiés | Section installation |

### Règle d'Application

```
┌─────────────────────────────────────────────────────────────────┐
│  RÈGLE: Chaque page DOIT contenir au minimum:                   │
│  • Critère #1 (Fiabilité) — TOUJOURS                            │
│  • Critère #5 (AI estimation) — CTA visible                     │
│  • 3 autres critères minimum                                    │
│                                                                 │
│  PAGES SOLUTION: Tous les 7 critères doivent être présents     │
└─────────────────────────────────────────────────────────────────┘
```

### Exemples d'Application

**Homepage:**
- Critère #1: Hero subheadline "5 Years Without a Single Breakage"
- Critère #5: CTA principal "Get Instant AI Estimate"
- Tous les autres: Sections dédiées

**Case Study:**
- Critère #1: Résultats documentés (failure rate, maintenance reduction)
- Critère #5: CTA dans les résultats
- Autres: Mentionnés dans le récit du projet

## 3. Architecture CTA

### Hiérarchie CTA (3 Niveaux)

```
NIVEAU 1: Solution Page
━━━━━━━━━━━━━━━━━━━━━━━
CTA PRINCIPAL: "See Our Products" → Catalogue
CTA SECONDAIRE: "Get AI Estimate" → AI Tool

NIVEAU 2: Catalogue
━━━━━━━━━━━━━━━━━━━
CTA: "View Details" → Product Detail Page

NIVEAU 3: Product Detail Page
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CTA PRINCIPAL: "Get AI Instant Quote" → AI Quote Flow
CTA SECONDAIRE: "Book a Call" → Calendar Booking
```

### Règles CTA

| Règle | Description |
|-------|-------------|
| **Unicité** | 1 CTA principal visible par écran |
| **Couleur** | Principal = accent color, Secondaire = outline |
| **Position** | Above the fold + fin de page |
| **Wording** | Action-oriented, pas "Submit" ou "Next" |

### CTA Standards

| Context | CTA Principal | CTA Secondaire |
|---------|---------------|----------------|
| Solution Page | See Our Products | Get AI Estimate |
| Product Detail | Get AI Instant Quote | Book a Call |
| Case Study | Start Your Project | View All Case Studies |
| Wizard (fin) | Get My Free Estimate | — |
| Supporting | Get AI Estimate | Contact Us |

## 5. Méthodologie de Rédaction

### Standards de Contenu

| Règle | Description |
|-------|-------------|
| **Paragraphes** | Minimum 3-5 phrases par paragraphe |
| **Sections** | Minimum 150-200 mots de contenu |
| **Jamais** | Section avec seulement 1-2 lignes |
| **Tonalité** | Professionnelle, directe, orientée bénéfices |

### Structure Standard des Sections

```
1. TITRE (H2/H3)
   - Clair, orienté bénéfice

2. COPY PRINCIPAL
   - 150-300 mots
   - Problème → Solution → Bénéfice
   - Données concrètes quand possible

3. ÉLÉMENTS VISUELS
   - Tableaux comparatifs
   - Listes à puces
   - Callout boxes

4. CTA (si applicable)
```

### Ton et Voix

| Caractéristique | Description |
|-----------------|-------------|
| **Professionnel** | Pas de slang, pas d'emojis excessifs |
| **Direct** | Phases simples, pas de jargon |
| **Bénéfice** | Focus sur ce que le client gagne |
| **Crédible** | Données vérifiables, pas d'hyperboles |

### Mots à Éviter

| Éviter | Remplacer par |
|--------|---------------|
| "Amazing", "Incredible" | "Reliable", "Proven" |
| "Cheap" | "Competitive pricing" |
| "Best" | "Industry-leading" |
| "Revolutionary" | "Engineered for..." |

## 6. Contraintes Techniques

### URLs

| Type | Pattern |
|------|---------|
| Industry | `/hotels`, `/real-estate`, `/senior-living` |
| Solution | `/led-mirrors`, `/smart-mirrors`, etc. |

| Case Study | `/case-studies/[case-study-slug]` |
| Supporting | `/about`, `/contact`, `/faq`, etc. |
| Wizard | `/estimate` |

### Responsive Design

- Mobile-first pour le wizard
- Desktop-first pour les pages de contenu
- Breakpoints: 640px, 768px, 1024px, 1280px

### Performance

| Métrique | Cible |
|----------|-------|
| LCP | < 2.5s |
| FID | < 100ms |
| CLS | < 0.1 |

### Accessibilité

- WCAG 2.1 AA minimum
- Alt text sur toutes les images
- Contraste minimum 4.5:1
- Navigation clavier

## 7. Frameworks de Conversion

### Les 7 Frameworks Utilisés

Ce projet intègre 7 frameworks de conversion éprouvés:

| # | Framework | Application |
|---|-----------|-------------|
| **1** | **MECLABS** | Value proposition, conversion sequence |
| **2** | **Hormozi** | Offer structure, value stacking |
| **3** | **Dan Kennedy** | Direct response, urgency |
| **4** | **Eugene Schwartz** | Awareness levels, copy stages |
| **5** | **StoryBrand** | Hero journey, clear messaging |
| **6** | **PAS** | Problem-Agitate-Solution |
| **7** | **AIDA** | Attention-Interest-Desire-Action |

### Application MECLABS (Principal)

```
MECLABS Conversion Sequence:
━━━━━━━━━━━━━━━━━━━━━━━━━━

1. MOTIVATION (M)
   - Visitor arrives with intent

2. VALUE PROPOSITION (V)
   - "Premium mirrors that last 5+ years"

3. INCENTIVE (I)
   - "Free AI estimate in 60 seconds"

4. FRICTION (F)
   - Minimize: typing, steps, uncertainty

5. ANXIETY (A)
   - Reduce: "No commitment", "No sales call"

CONVERSION = (M + V + I) - (F + A)
```

### Application du Wizard

| Framework | Application dans le Wizard |
|-----------|---------------------------|
| **Cialdini - Commitment** | 6 micro-commitments avant formulaire |
| **Cialdini - Reciprocity** | Valeur partielle avant demande |
| **Loss Aversion** | "Your estimate will be ready..." |
| **Anchoring** | Prix complet montré avant discount |
